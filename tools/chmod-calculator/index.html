Linux / chmod

chmod 权限计算器

勾选权限矩阵、输入八进制数字或符号表示（-rwxr-xr-x），三处任改一处其余实时同步，并生成可复制的 chmod 命令。

权限矩阵

对象读 r (4)写 w (2)执行 x (1)
所有者 (u)
所属组 (g)
其他人 (o)
特殊位：

纯本地计算，八进制支持 3 位（755）或 4 位带特殊位（4755）。

命令

chmod 000 file

点击上方命令即可复制到剪贴板。

常用权限速查

数字符号说明
644rw-r--r--普通文件默认：所有者可读写，其他人只读
755rwxr-xr-x目录与脚本常用：所有者全权，其他人可读可执行
600rw-------私密文件：仅所有者可读写（SSH 私钥必须）
700rwx------私密目录：仅所有者可进入访问
775rwxrwxr-x协作目录：同组成员可写入
664rw-rw-r--协作文件：同组成员可编辑
777rwxrwxrwx任何人可读写执行——高风险，生产环境禁用

点击任意一行即可套用该权限。

关于chmod权限计算器

chmod 是 Linux / macOS 修改文件权限的命令，权限由所有者、所属组、其他人三组，各自的读（4）、写（2）、执行（1）组合而成，累加即得 755、644 这类八进制数字。本工具在勾选矩阵、八进制数字、符号表示（-rwxr-xr-x）之间三向实时换算，支持 setuid、setgid、sticky 特殊位（4 位写法如 4755），并内置常用权限速查表与一键复制的 chmod 命令，全部在浏览器本地完成。

常见问题

chmod 755 和 644 有什么区别？

755 = rwxr-xr-x，所有者可读写执行，组和其他人可读可执行，适合目录和可执行脚本；644 = rw-r--r--，所有者可读写，其他人只读且不可执行，适合普通文件（网页、配置、图片）。经验法则：目录 755、文件 644。

八进制权限数字是怎么算出来的？

每组权限用读=4、写=2、执行=1 累加：rwx=4+2+1=7，r-x=4+1=5，r--=4。三组从左到右分别是所有者、所属组、其他人，拼起来就是 755。第 4 位（写在最前）表示特殊位：setuid=4、setgid=2、sticky=1，例如 4755。

为什么不建议用 chmod 777？

777 意味着系统上任何用户都能读取、修改和执行该文件，恶意程序或其他用户可以随意篡改内容、植入代码，是常见的安全隐患。遇到权限问题应先用 ls -l 查明所有者和组，再用 chown/chmod 精确授权，而不是一律 777。

关于cURL命令解析

    
curl 是最常用的命令行 HTTP 客户端，浏览器开发者工具和各类 API 文档都习惯以 curl 命令的形式给出请求示例。本工具在浏览器本地解析 curl 命令行：自带 shell 风格分词器，支持单引号、双引号、反斜杠转义与换行续行，识别 -X、-H、-d、-F、-u、-b、--json、-G、-k、-L、--compressed 等常用参数，解析后以表格展示方法、URL、Query 参数、Headers 和 Body，并一键生成等价的 JavaScript fetch 代码和 Python requests 代码，命令内容不会上传到任何服务器。

    
常见问题

    
如何从浏览器拿到一条 curl 命令？
打开开发者工具的 Network 面板，找到目标请求后右键选择 Copy → Copy as cURL 即可复制完整命令，粘贴到本工具就能还原出请求结构并生成代码。注意 Windows 下请选 cURL (bash) 而不是 cURL (cmd)，两者引号规则不同。

    
生成的 fetch 代码为什么没有 -k 和 --compressed 的效果？
浏览器的 fetch 由浏览器统一管理 TLS 校验和内容压缩，无法在代码里跳过证书校验（-k）或手动声明压缩（--compressed），工具会以注释提示。Python requests 则可以用 verify=False 对应 -k，压缩由 requests 自动处理。

    
支持多条 curl 命令或管道吗？
每次只解析一条 curl 命令，命令末尾的 | jq 之类管道内容会被当作无法识别的参数在提示中列出，不影响主体解析。多条命令请分开粘贴解析。

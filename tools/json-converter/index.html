关于JSON转YAML

    
拿到一段接口返回的 JSON，经常需要手写对应的类型定义或换一种格式——本工具一键完成：转 YAML（写 K8s、CI 配置）、转 TypeScript interface（嵌套对象自动拆分接口）、转 Go struct（字段自动转大驼峰并带 json tag）、转 CSV（对象数组导出表格）。省去手敲类型的时间，也避免抄写错漏。

    
常见问题

    
JSON 和 YAML 怎么选？
JSON 是机器交换格式：语法严格、解析快、无注释；YAML 是人类编辑格式：支持注释、层级靠缩进、写起来简洁，但缩进错误不易察觉。接口传输用 JSON，需要人手维护的配置用 YAML。

    
生成的 TypeScript 类型可以直接用吗？
可以作为起点：结构、嵌套接口和可选引号键名都会正确生成。注意类型基于当前样本推断，如果某字段可能为 null 或有多种类型，需要手动改成联合类型（如 string | null）。

    
转 CSV 时报错"要求输入为对象数组"？
CSV 是二维表格，只有形如 [{"a":1},{"a":2}] 的对象数组才能转换（每个对象是一行、键是列名）。单个对象或嵌套过深的结构请先在外面包一层数组或拍平。

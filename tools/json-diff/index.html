关于JSON对比

本工具对两段 JSON 做结构化差异对比（deep diff）：先解析再递归比较，逐条列出新增、删除、修改和类型变化的字段，每条差异都给出完整 JSON 路径（如 config.items[2].name），一眼定位改动位置。因为比较的是解析后的数据结构，key 顺序、缩进和空格的不同不会被误报为差异；数组默认按索引对比，也可勾选忽略顺序。适合对比接口两次返回、配置文件版本和数据快照，全程本地处理不上传。

常见问题

和文本 diff 有什么区别？

文本 diff 逐行比较字符，key 顺序变了、缩进不同都会报差异；JSON 结构化对比先解析再比较数据本身，只关心字段和值的真实变化，并直接给出字段路径，排查接口和配置改动更精准。

支持多大的 JSON？

解析和递归对比都在浏览器内存中完成，几 MB 的 JSON 毫无压力；上限取决于浏览器内存，几十 MB 的超大文件建议先拆分或用命令行工具处理。

数据会上传到服务器吗？

不会。解析和对比全部在浏览器本地用 JavaScript 完成，内容不经过任何服务器，可以放心粘贴包含内部字段的接口返回和配置。

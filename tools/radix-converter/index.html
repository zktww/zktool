关于进制转换

进制转换是编程和数字电路中的高频操作：调试位掩码要看二进制，读文件权限要看八进制，内存地址和颜色值离不开十六进制。本工具提供二进制、八进制、十进制、十六进制四栏实时联动互转，任意一栏输入其余自动更新；基于 BigInt 任意精度整数实现，超大整数和负数都能精确换算，输入兼容 0x、0b、0o 前缀；还支持 2-36 之间的自定义进制（如 Base32、Base36），并附带 bit 长度、字节数、2 的幂判断与常用数值对照表。所有计算均在浏览器本地完成，数据不会上传。

常见问题

支持多大的数字？会不会精度丢失？

本工具基于 BigInt 任意精度整数实现，几百上千位的大整数也能精确换算，不会像 JavaScript Number 那样在超过 2^53 后丢失精度。负数在最前面加 - 即可，暂不支持小数部分的进制转换。

0x、0b、0o 这些前缀是什么意思？

它们是编程语言里常见的进制字面量前缀：0x 表示十六进制（如 0xff）、0b 表示二进制（如 0b1010）、0o 表示八进制（如 0o777）。在对应输入框中粘贴带前缀的数值会被自动识别，无需手动删除前缀。

自定义进制为什么最大只到 36？

因为常规写法可用的字符是 0-9 十个数字加 a-z 二十六个字母，总共 36 个，所以逐位记数法最大到 36 进制。Base36 常用于生成短链接、邀请码等紧凑编号；本工具输入时字母大小写不敏感，输出统一为小写。

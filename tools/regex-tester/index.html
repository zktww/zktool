关于正则表达式测试

    
本工具实时运行 JavaScript 正则表达式（RegExp），输入即测：匹配内容在原文中高亮显示，每处匹配的捕获组（含 (?...) 命名分组）逐条列出，还能预览 replace 替换结果。正则在独立的 Web Worker 中执行并带 1 秒超时保护，即使写出灾难性回溯的表达式也不会卡死页面，适合放心调试复杂正则。

    
常见问题

    
正则的 g、i、m 这些 flag 是什么意思？
g 全局匹配（找出所有而非第一个）、i 忽略大小写、m 多行模式（^ $ 匹配每行首尾）、s 让 . 匹配换行、u Unicode 模式。本工具在 flags 框直接填写组合即可。

    
什么是灾难性回溯？
形如 (a+)+ 的嵌套量词在匹配失败时会产生指数级回溯尝试，让正则引擎"卡死"。本工具的 1 秒超时会中止这类执行并提示；修复思路是消除嵌套量词或改用更精确的字符类。

    
测试通过的正则在 Python/Java 里不生效？
各语言正则方言有差异：本工具运行的是 JavaScript 方言，Python 的命名分组是 (?P...)、Java 字符串里 \ 需要双写。语法主体通用，迁移时注意这些细节。

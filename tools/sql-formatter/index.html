关于SQL格式化

    
从日志或代码里拿到的 SQL 往往挤成一行，读起来很吃力。本工具一键把 SQL 格式化为规范缩进的多行形式，支持 MySQL、PostgreSQL、SQLite、SQL Server、Oracle、Hive、Spark、BigQuery 等 9 种方言的语法差异，关键字大小写（UPPER/lower）和缩进风格可调；也能反向把多行 SQL 压缩成单行（保留字符串内容、移除注释），方便塞进配置或命令行。

    
常见问题

    
格式化会改变 SQL 的执行结果吗？
不会。格式化只调整空白、换行和关键字大小写，语句的语义完全不变；字符串字面量里的内容（包括其中的空格）会原样保留。

    
为什么要选择 SQL 方言？
各数据库语法有差异：反引号是 MySQL 的标识符引用、TOP 是 SQL Server 的、LATERAL VIEW 是 Hive 的。选对方言才能正确解析这些专有语法，不确定时选"标准 SQL"通常也能工作。

    
支持格式化存储过程或很长的 SQL 吗？
支持常见的 DML/DDL 和多语句脚本；数千行的 SQL 也能处理。极复杂的存储过程语法（如深度嵌套的流程控制）可能解析失败，此时会保留原文并提示错误。

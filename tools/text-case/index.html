关于驼峰命名转换

    
改字段名和变量名是开发日常：数据库用 snake_case、JavaScript 用 camelCase、CSS 类名用 kebab-case、常量用 CONSTANT_CASE。本工具支持这些命名风格的批量互转（每行一条），能正确处理 HTTPResponse 这类连续大写缩写的边界；另附行级处理：去重、按拼音/字母排序、去空行、倒序，以及字符/词/行统计，转换结果可以一键回填继续叠加操作。

    
常见问题

    
什么时候用驼峰、什么时候用下划线？
看语言与场景约定：JavaScript/Java 变量用 camelCase，类名用 PascalCase；Python 变量和数据库字段惯用 snake_case；CSS 类名和 URL 用 kebab-case；常量用 CONSTANT_CASE。同一项目保持一致最重要。

    
HTTPResponse 这种缩写怎么正确拆分？
本工具按"连续大写 + 后跟小写"识别缩写边界：HTTPResponseCode 会拆成 HTTP / Response / Code，转 snake_case 得到 http_response_code，而不是 h_t_t_p 这种错误结果。

    
能一次转换多个字段名吗？
可以，每行一条批量处理。比如把整张表的列名粘进来一键转驼峰，再用"用结果替换输入"叠加去重或排序。

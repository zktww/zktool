关于时区会议规划

跨时区协作最头疼的就是约时间：北京下午三点是伦敦早上几点？纽约同事这会儿是不是已经下班？本工具基于浏览器内置的 Intl API 做时区换算——自动处理各地夏令时切换，无需查表。选定一个基准日期时间和基准城市后，对照表会列出你添加的每个城市对应的当地时间、与基准的时差，并用绿/黄/红圆点标注当地是否处于 9:00-18:00 工作时间；下方的 24 小时条形视图把一整天的重叠情况铺开，绿色格子越多的列越适合开会。全部计算在浏览器本地完成，城市选择仅存在本机。

常见问题

夏令时会自动处理吗？

会。本工具使用 IANA 时区标识（如 Europe/London、America/New_York）配合浏览器 Intl API 计算，夏令时的开始与结束由浏览器内置的时区数据库自动处理，伦敦冬令时 UTC+0、夏令时 UTC+1 这类切换无需手动调整。

时差为什么会出现半小时？

并非所有时区都是整小时偏移：印度标准时间是 UTC+5:30，与北京相差 2.5 小时；还有尼泊尔 UTC+5:45 这类 45 分钟偏移的时区。工具按分钟精度计算时差，会如实显示 +5.5h 这样的半小时差。

怎样快速找到大家都方便的会议时间？

看 24 小时条形视图：每列对应基准城市的一个小时，找绿色（工作时段）格子最多的列即可。中国与欧洲的重叠通常在北京时间 16:00-18:00，与美国西海岸则集中在北京时间上午 8:00-10:00 前后。

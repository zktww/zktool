关于URL编码解码

    
URL 中的中文、空格和特殊符号必须经过百分号编码（Percent-Encoding）才能安全传输，这就是链接里 %E4%B8%AD 这类字符的由来。本工具提供 encodeURIComponent 编码与解码，并能把完整 URL 或查询字符串一键拆解为参数表格和 JSON 对象（同名参数自动合并为数组），适合排查接口传参、还原分享链接和分析重定向地址。

    
常见问题

    
URL 里的 %20 和 + 都是空格吗？
%20 在任何位置都表示空格；+ 只在查询字符串（application/x-www-form-urlencoded 表单编码）中表示空格，在路径里就是加号本身。本工具解码时可勾选"+ 视为空格"按需处理。

    
encodeURI 和 encodeURIComponent 有什么区别？
encodeURIComponent 会编码所有保留字符（包括 / ? & =），适合编码单个参数值；encodeURI 保留 URL 结构字符，适合编码整个网址。给参数编码用前者，否则 & 会破坏参数结构。

    
解码报错 URIError: malformed 是什么原因？
文本中存在不完整的百分号序列（如单独的 % 或 %ZZ）。常见于被截断的链接或双重编码的内容；可以先检查是否需要连续解码两次，或修正残缺的 % 序列。

关于YAML转换与校验

    
YAML 是 Kubernetes、Docker Compose、GitHub Actions 等场景最常用的配置格式，靠缩进表达层级，写错一个空格就可能解析失败。本工具基于 js-yaml 在浏览器本地完成 YAML 与 JSON 的双向互转：YAML 转 JSON 支持 2/4 空格缩进，JSON 转 YAML 自动整理为规范缩进；输入时实时校验语法，出错会精确提示行号和列号，配置文件不会上传到任何服务器。

    
常见问题

    
YAML 校验报错但看不出哪里错了怎么办？
关注提示中的行号和列号，最常见的原因是缩进混用了 Tab 和空格（YAML 只允许空格）、冒号后面缺少空格、字符串里包含 : 或 # 等特殊字符没有加引号。定位到对应行逐一排查即可。

    
YAML 和 JSON 有什么区别？可以无损互转吗？
JSON 是 YAML 的子集，任何合法 JSON 都是合法 YAML，所以 JSON 转 YAML 总是可行。反向转换时，YAML 特有的注释、锚点引用（&/*）和多文档分隔（---）会在转成 JSON 后丢失，数据本身不会丢。

    
为什么 YAML 里的 no、on、1.0 转成 JSON 后变了类型？
YAML 会把未加引号的 no/off 解析为布尔值 false，on/yes 解析为 true，1.0 解析为数字，这是 YAML 规范的隐式类型转换。如果希望保留为字符串，请在 YAML 中给值加上引号，例如 country: "NO"。
